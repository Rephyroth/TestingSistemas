2.0 que es prueba de software
	verificacion y  validacion de los requerimetos funcionales
	verificacion es comparar caso de prueba con caso de uso
2.1 Atributos de calidad de un sistema 
	 se hace para encontrar errores mediabte el uso de procesos, procedimientos y herramientas
	 
2.2 Tipos de pruebas de software 
	Funcional 
		Manual: el operario se encarga de la prueba
				lo que es sistema debe de hacer o no debe de hacer

		Automatizado: algoritmo que se dedica a hacer las pruebas
				se utilizan Frameworks 
	No Funcional
		Desempeño: validar el uso de recursos de un programa con un cierto volumen de transacciones
			se monitorea tidos los recursos que utiliza la maquina  
		Seguridad: validar los puntos debiles del programa
2.3 Ciclos de vuda de prueba de software
	-análisis
	-Planeación
	-Diseño
	-Ambiente
	-Ejecución
	-UAT
	-Cierre
2.3 Que es un caso de Prueba
	Entradas: Dato o evento externo que se inyecta en el sistema
	Precondiciones: Estado en que tiene que estar el sistema para capturar la entrada
	Salida: Dato que sale del sistema o mensaje de resultado
	Pos Condicion: Estado en que se queda el sistema despues de ejecutar la entraday precondicion
2.4 Datos de prueba 
	Son datos que se recquieren para hacer la prueba 
	cada Test Case requiere datos de entrada 
	Tipos de datos: 
		Estaticos
		Dinamicos
